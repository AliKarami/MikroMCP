# MikroMCP v1.5 — Container Depth, Network Gap-Fill & Diagnostics

**Date:** 2026-05-27
**Status:** Approved

---

## Goal

Complete the container management surface (config, env vars, mounts), add bandwidth and URL fetch diagnostics, and fill the remaining gaps in DNS write, file delete, IPSec policy write, WireGuard interface management, interface lists, and PPP profiles.

**Tool count: 99 → 117** (18 new tools)

## Architecture

Each new file follows the established pattern: Zod `.strict()` schema, idempotency check before any write, dry-run on every write action, `enrichError` in every catch, `MikroMCPError` re-thrown as-is. Modified files receive new `ToolDefinition` objects appended at the bottom and exported via the existing array.

`src/domain/tools/index.ts` registers four new arrays:
```typescript
import { containerConfigTools } from "./container-config-tools.js";
import { diagnosticsTools } from "./diagnostics-tools.js";
import { interfaceListTools } from "./interface-list-tools.js";
import { pppTools } from "./ppp-tools.js";
```

Test files mirror the source split. Modified tool files get new `describe` blocks appended to their existing test file. New tool files get fresh test files under `test/unit/domain/`.

## Tool Details

### `manage_dns_settings` — `ip/dns` (POST)

**Singleton** — no ID or idempotency key. Reads current settings first, diffs against requested values, returns `no_change` if nothing differs.

Parameters:
- `routerId` (required)
- `servers` — comma-separated upstream DNS server IPs
- `allowRemoteRequests` — bool; allow the router to answer DNS queries from the network
- `maxUdpPacketSize` — int, 512–65535
- `cacheMaxTtl` — string duration (e.g. `"1d"`)
- `cacheSize` — int KB
- `dryRun` — bool, default false

`readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true`.
`snapshotPaths: ["ip/dns"]`

---

### `delete_file` — `file/{id}` (DELETE)

Looks up the file by `name` field in `GET /file`. Returns `not_found` gracefully if no matching file exists (idempotent remove). `dryRun` previews what would be deleted.

Parameters:
- `routerId` (required)
- `name` — filename on the router (required)
- `dryRun` — bool, default false

`readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true`.

---

### `manage_ipsec_policy` — `ip/ipsec/policy`

Idempotency key: composite `srcAddress` + `dstAddress` + `tunnel`. Actions: `add`, `remove`, `enable`, `disable`.

Parameters:
- `routerId`, `action` (required)
- `srcAddress` — source CIDR (required)
- `dstAddress` — destination CIDR (required)
- `tunnel` — bool, default false; whether this is a tunnel-mode policy
- `ipsecAction` — `"encrypt"` | `"discard"` | `"none"` (required for add)
- `level` — `"require"` | `"use"` | `"unique"` (optional, default `"require"`)
- `saSourceAddress` — SA source IP for tunnel mode
- `saDstAddress` — SA destination IP for tunnel mode
- `dryRun` — bool, default false

`readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true`.
`snapshotPaths: ["ip/ipsec/policy"]`

---

### `manage_wireguard_interface` — `interface/wireguard`

Idempotency key: `name`. Actions: `add`, `remove`, `enable`, `disable`. RouterOS generates the private key on create — it is never passed in or returned. The public key returned by RouterOS after creation is included in the `structuredContent`.

Parameters:
- `routerId`, `action` (required)
- `name` — interface name (required)
- `listenPort` — int 1–65535 (optional; RouterOS picks one if omitted)
- `mtu` — int 1280–65535 (optional, default 1420)
- `comment` — optional string
- `dryRun` — bool, default false

`readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true`.

---

### `get_container_config` — `container/config`

Read-only singleton. Returns registry URL, RAM limit, veth interface name.

`readOnlyHint: true`, `destructiveHint: false`, `idempotentHint: true`.

---

### `manage_container_config` — `container/config` (POST/PUT)

Singleton write. Reads current config first, diffs, returns `no_change` if nothing differs. `dryRun` returns the diff without applying.

Parameters:
- `routerId` (required)
- `registryUrl` — container registry URL
- `ramHighMb` — RAM high-water mark in MB
- `veth` — veth interface name
- `dryRun` — bool, default false

`readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true`.
`snapshotPaths: ["container/config"]`

---

### `list_container_envs` — `container/envs`

Filter by `name` (container name). Returns all env var entries or those belonging to a specific container.

Parameters: `routerId`, `name` (optional filter), `limit` (default 100).

`readOnlyHint: true`.

---

### `manage_container_env` — `container/envs`

Idempotency key: `name` + `key`. Actions: `add`, `remove`. `add` returns `already_exists` if the entry exists with the same value; throws `CONFLICT` if it exists with a different value.

Parameters:
- `routerId`, `action` (required)
- `name` — container name (required)
- `key` — env var name (required)
- `value` — env var value (required for add)
- `dryRun` — bool, default false

`readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true`.

---

### `list_container_mounts` — `container/mounts`

Filter by `name` (mount name or container name). Returns mount definitions with `src`, `dst`, and associated container.

Parameters: `routerId`, `name` (optional filter), `limit` (default 100).

`readOnlyHint: true`.

---

### `manage_container_mount` — `container/mounts`

Idempotency key: `name` (mount name). Actions: `add`, `remove`. `add` returns `already_exists` if a mount with the same name exists with matching `src`/`dst`; throws `CONFLICT` if name exists with different paths.

Parameters:
- `routerId`, `action` (required)
- `name` — mount name (required, idempotency key)
- `src` — host source path (required for add)
- `dst` — container destination path (required for add)
- `dryRun` — bool, default false

`readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true`.

---

### `bandwidth_test` — `tool/bandwidth-test` (execute)

Runs a RouterOS bandwidth test from the router to a remote host. `readOnlyHint: true` (no config is changed). Duration capped at 30 s to prevent long-running calls. The remote host must be running the RouterOS bandwidth-test server (`btest`) — this tool does not test against arbitrary TCP/UDP endpoints.

Parameters:
- `routerId`, `address` (required)
- `protocol` — `"tcp"` | `"udp"` (default `"tcp"`)
- `direction` — `"send"` | `"receive"` | `"both"` (default `"both"`)
- `duration` — int seconds, 1–30 (default 5)

Returns: `txMbps`, `rxMbps`, `lostPackets` (UDP only), `routerId`.

`readOnlyHint: true`, `destructiveHint: false`, `idempotentHint: false`.

---

### `fetch_url` — `tool/fetch` (execute)

Sends an HTTP/HTTPS request from the router. Response body capped at 64 KB with a `[TRUNCATED]` marker. `outputFile` saves to router filesystem instead of returning body inline.

Parameters:
- `routerId`, `url` (required)
- `method` — `"GET"` | `"POST"` (default `"GET"`)
- `httpData` — request body for POST (optional)
- `outputFile` — router file path to save response body (optional; if provided, body is not returned inline)

Returns: `statusCode`, `body` (string, capped at 64 KB, omitted when `outputFile` is used), `outputFile` (when provided).

`readOnlyHint: true`, `destructiveHint: false`, `idempotentHint: false`.

---

### `list_connections` — `ip/firewall/connection`

Read-only. Paginate client-side (RouterOS connection table does not support server-side filtering via REST). Filter by `srcAddress`, `dstAddress`, `protocol` applied client-side after fetching.

Parameters:
- `routerId` (required)
- `srcAddress` — optional filter (substring match on src address)
- `dstAddress` — optional filter (substring match on dst address)
- `protocol` — optional exact match
- `limit` — default 100, max 500

`readOnlyHint: true`, `destructiveHint: false`, `idempotentHint: true`.

---

### `list_interface_lists` — `interface/list`

Returns all interface lists with name and comment. No filter needed (typically few lists).

Parameters: `routerId`, `limit` (default 100).

`readOnlyHint: true`.

---

### `manage_interface_list` — `interface/list`

Idempotency key: `name`. Actions: `add`, `remove`. Removing a list that has members is blocked by RouterOS — the error is surfaced via `enrichError`. `dryRun` supported.

Parameters:
- `routerId`, `action` (required)
- `name` — list name (required)
- `comment` — optional string
- `dryRun` — bool, default false

`readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true`.

---

### `manage_interface_list_member` — `interface/list/member`

Idempotency key: composite `list` + `interface`. Actions: `add`, `remove`. `add` returns `already_exists` if the membership exists. `remove` returns `not_found` gracefully.

Parameters:
- `routerId`, `action` (required)
- `list` — interface list name (required)
- `interface` — interface name to add/remove (required)
- `comment` — optional string
- `dryRun` — bool, default false

`readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true`.

---

### `list_ppp_profiles` — `ppp/profile`

Filter by `name`. Returns all PPP profiles including the built-in `default` and `default-encryption`.

Parameters: `routerId`, `name` (optional exact filter), `limit` (default 100).

`readOnlyHint: true`.

---

### `manage_ppp_profile` — `ppp/profile`

Idempotency key: `name`. Actions: `add`, `update`, `remove`. `update` returns `no_change` when requested values match current values. Built-in profiles (`default`, `default-encryption`) cannot be removed — RouterOS returns an error which is surfaced via `enrichError`. `dryRun` supported for all actions.

Parameters:
- `routerId`, `action` (required)
- `name` — profile name (required)
- `localAddress` — local IP assigned to router end of PPP link
- `remoteAddress` — IP or pool name assigned to client
- `dnsServer` — DNS server pushed to client
- `rateLimit` — rate limit string (e.g. `"10M/10M"`)
- `sessionTimeout` — duration string (e.g. `"1h"`)
- `comment` — optional string
- `dryRun` — bool, default false

`readOnlyHint: false`, `destructiveHint: false`, `idempotentHint: true`.
`snapshotPaths: ["ppp/profile"]`

## Error Handling

All tools follow the existing rules:
- Handler body wrapped in `try/catch`; `enrichError(err, context)` called in catch
- `MikroMCPError` re-thrown as-is; never double-wrapped
- Domain errors (`NOT_FOUND`, `CONFLICT`, `VALIDATION`) thrown directly as `MikroMCPError`
